코드 규칙

HTML 규칙

적절한 태그 및 속성 사용

  • 클릭했을 때 이벤트가 발생하는 경우 button 태그를 사용한다.
  • 클릭했을 때 다른 페이지로 넘어가는 경우 a 태그를 사용한다.
  • a 태그 안에 button 태그를 넣는 등 인터랙티브 요소에 인터랙티브 요소를 넣지 않도록 한다.
  • 단락이 아닌 경우, 단순히 block 속성을 사용하기 위해서 p 태그를 사용하지 않는다.
  • heading 태그는 레벨 순서를 지켜 작성한다.
  • button 태그에는 type 을 명시한다.
  • 레이아웃을 표현하기 위하여 표를 사용하지 않는다.
  • 리스트는 table 태그가 아닌 ul 태그를 사용한다.

웹접근성

  • img 요소의 alt 속성은 반드시 제공되어야 하며, 의미없는 이미지에 한하여 빈 값을 허용한다.
  • input 요소는 label 과 연결하거나 title 값을 지정하여야 한다.
  • 분리된 입력폼은 각각 title 값을 지정하여야 한다.
  • 적절한 label 텍스트 또는 title 를 사용하여야 한다.
  • 키보드 탭키 이동으로 접근이 불가능한 내용이 없어야 한다.

ID와 클래스

  • ID는 개발을 위해서만 사용하고, 퍼블리싱 작업시에는 클래스를 활용한다.
  • 불가피한 경우를 제외하고 클래스명은 직관적으로 짓는다.
  • 체인 선택자의 개수는 3개까지 권장한다.

들여쓰기

  • 1탭 들여쓰기 하며, 1탭의 크기는 공백 4칸으로 설정한다.

SCSS 규칙

scss 임포트

  • 파일 구조는 컴포넌트 형식으로 파일을 나누어 작업한다.
  • 컴포넌트 형식으로 파일을 나눈 후, 하나의 스타일시트에 임포트하는 방식으로 진행한다.

속성

  • 모든 속성은 영문 소문자로만 작성한다.
  • 약식으로 사용할 수 있는 경우 약식으로 작성한다.
  • 컬러값은 단축 가능하다면 단축한 값으로 기재한다. (ex. #fff)
  • 스타일이 지정된 이름은 사용하지 않는다. (스타일이 변경되면 클래스명까지 변경을 해야된다.)
                    
                        /* 사용 X */
                        .mt-10 {margin-top: 10px;}
                        .ml-10 {margin-left: 10px;}
                        .red-box {background-color: red;}

선택자 및 우선순위

  • 공통 선택자(*)는 웹페이지의 성능 저하를 가져오므로 사용하지 않는다.
  • 태그는 언제든지 다른 태그로 변경될 수 있으므로 태그 선택자로 스타일을 작성하지 않는다.
  • !important 는 불가피한 상황을 제외하고는 사용하지 않는다.
  • 인라인 스타일의 우선 순위가 높으므로 인라인 스타일은 사용하지 않는다.
  • 선택자를 다수 작성할수록 우선 순위가 높아지므로 선택자는 가능하다면 최소 단위 / 루트 단위로 진행한다.
  • 클래스명이 자주 쓰이는 명칭일 경우 선택자를 하나, 두 개 정도 추가 작성한다.
  • div 태그에는 클래스명을 지정한다.

세미콜론

  • 마지막 선언된 속성에도 세미콜론(;)을 사용한다.

들여쓰기

  • 1탭 들여쓰기 하며, 1탭의 크기는 공백 4칸으로 설정한다.

공백의 사용

  • 속성 선언시 콜론 (:) 뒤에 한칸의 공백을 사용한다.
                    
                        .class-name {color: #ddd;font-size: 1.4rem;}

반복

  • 반복적으로 사용되는 스타일의 경우 mixin scss의 함수를 활용할 수 있다.
  • 반복적으로 사용되는 값(컬러 색상, 값…)의 경우 변수를 선언하여 활용할 수 있다.

중첩

  • 중첩을 활용하여 상위 선택자의 반복을 쓰지 않는다.
                    
                        /* 사용 X */
                        .cont ul li .flex-item.login-phone {background-color: #fcfcfc;}
                        .cont ul li .flex-item.login-phone .select {color: #ddd;font-size: 1.4rem;}
                        
                        /* 사용 O */
                        .cont {
                            ul {
                                li {
                                    .flex-item {
                                        &.login-phone {background-color: #fcfcfc;
                                            .select {color: #ddd;font-size: 1.4rem;}
                                        }
                                    }
                                }
                            }
                        }

네임 규칙

표기법

종류 구분 사용법 예시
ID Selector 카멜 표기법 처음단어의 첫문자는 소문자, 다음단어의 첫문자는 대문자 userName, userAge, phoneNumber
Class Selector 케밥 표기법 다음단어와 '-'(대시)로 구분 user-name, user-age, phone-number
Image 스네이크 표기법 다음단어와 '_'(언더바)로 구분 user_name, user_age, phone_number
파일 및 폴더 스네이크 표기법 다음단어와 '_'(언더바)로 구분 total_sch_01, edu_01

ID 선택자

  • ID는 개발을 위해서만 사용하고, 퍼블리싱 작업시에는 앵커 및 label 등의 연결하는 상황 등을 제외하고는 일반적으로 사용하지 않으므로 정리하여 기재하지 않는다.

class 선택자

  • 단어의 경우 직관적으로 알아볼 수 있도록 가능한 축약하지 않는다.
  • 체인 선택자의 개수는 3개까지 권장한다.
  • 클래스 예약어는 상황에 맞게 선택적으로 참고한다.
